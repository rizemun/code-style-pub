Code style

Файлы

  • PHP-код ОБЯЗАТЕЛЬНО следует заключать в полную версию (<?php ?>) тегов или укороченную (сокращённую запись echo) версию (<?= ?>) тегов и НЕДОПУСТИМО заключать ни в какие иные разновидности тегов.
  • Закрывающий тег ?> ДОЛЖЕН отсутствовать в файлах, содержащих только PHP-код.
  • Во всех файлах с PHP-кодом ДОЛЖЕН быть использован Unix-вариант переноса строк (Unix line feed, т.е. \n).
  • В конце каждого файла с PHP-кодом ДОЛЖНА быть одна пустая строка.
  • PHP-код ДОЛЖЕН быть представлен только в кодировке UTF-8 без BOM-байта.
  • Расширение файлов с PHP-кодом ОБЯЗАТЕЛЬНО должно быть .php
  • Названия файлов с кодом и имена каталогов ДОЛЖНЫ быть в «StudlyCaps» (каждое слово начинается с большой буквы, между словами нет разделителей).

Строки кода

  • СЛЕДУЕТ стараться, чтобы длина строки кода составляла 120 символов или менее; более длинные строки СЛЕДУЕТ разбивать на несколько отдельных строк, длина каждой из которых не превышала бы 120 символов.
  • В конце непустых строк НЕ ДОЛЖНО быть пробелов.
  • Пустые строки МОГУТ быть добавлены в код для повышения удобочитаемости и разделения блоков кода.
  • В одной строке НЕ ДОЛЖНО быть более одного выражения, заканчивающегося ;

Отступы

  • Для оформления отступов ДОЛЖНА использоваться табуляция.

Ключевые слова и константы true / false / null

  • Ключевые слова PHP ДОЛЖНЫ быть написаны в нижнем регистре.
  • Константы PHP true, false и null ДОЛЖНЫ быть написаны в нижнем регистре.

Классы

  • Каждый класс должен располагаться в отдельном файле и в пространстве имён с хотя бы одним верхним уровнем (именем производителя).
  • Под «классом» следует понимать также интерфейсы (interface) и примеси (trait).
  • Имена классов ДОЛЖНЫ быть объявлены с использованием т.н. «StudlyCaps» (каждое слово начинается с большой буквы, между словами нет разделителей).
  • Имя файла класса ДОЛЖНО совпадать с именем класса внутри.
  • Ключевые слова extends и implements ДОЛЖНЫ находиться на той же строке, на которой находится имя класса.
  • Открывающая фигурная скобка в определении класса ДОЛЖНА располагаться на текущей строке, а закрывающая фигурная скобка ДОЛЖНА располагаться на следующей строке после тела класса.
<?php
 
class Bar extends Foo {
    //код
}

Пространства имён

  • В случае наличия определения пространства имён, после него ДОЛЖНА располагаться одна пустая строка.
  • В случае наличия импорта пространств имён, он ДОЛЖЕН располагаться после определения пространства имён.
  • В случае наличия require, он должен располагаться после импорта пространств имён.
  • При реализации импорта каждое пространство имён ДОЛЖНО импортироваться отдельно (со своим ключевым словом use).
  • После блока импорта ДОЛЖНА быть одна пустая строка.
<?php
 
namespace Vendor\Package;
 
use FooClass;
use BarClass as Bar;
use OtherVendor\OtherPackage\BazClass;
 
// ... далее следует PHP-код ...

Константы

  • Константы классов ДОЛЖНЫ быть объявлены в верхнем регистре с использованием символа подчёркивания в качестве разделителя слов, например
const VERSION = '1.0';
const DATE_APPROVED = '2012-06-01';

Свойства

  • Свойства ДОЛЖНЫ именоваться в "under_score" (английские слова в идентификаторах разделяются подчеркиваниями).
private $database_name;
  • Область видимости ДОЛЖНА быть явно указана для каждого свойства.
  • СЛЕДУЕТ использовать как можно более закрытые модификаторы доступа (желательно не использовать public).
  • При определении свойства НЕ ДОЛЖНО применяться ключевое слово var.
  • В одном выражении НЕ ДОЛЖНО быть определено более одного свойства.
  • Одиночный знак подчёркивания в начале имени свойства НЕ СЛЕДУЕТ использовать как признак защищённой (protected) или приватной (private) области видимости.

Методы

  • Имена методов ДОЛЖНЫ быть объявлены с использованием т.н. «camelCase» (первое слово пишется в нижнем регистре, далее каждое слово начинается с большой буквы, а между словами нет разделителей).
private function debugInfo(int param1, string param1,stringparam2 = 'default') : int {
    //PHP-код
}
  • Область видимости ДОЛЖНА быть явно указана для каждого метода.
  • Одиночный знак подчёркивания в начале имени метода НЕ СЛЕДУЕТ использовать как признак защищённой (protected) или приватной (private) области видимости.
  • СЛЕДУЕТ стараться использовать как можно более закрытые модификаторы доступа.
  • После имени метода НЕ ДОЛЖНО быть пробела. Открывающая фигурная скобка ДОЛЖНА находиться на текущей строке, а закрывающая фигурная скобка ДОЛЖНА находиться на следующей за телом метода строке.
  • В списке аргументов методов НЕ ДОЛЖНО быть пробела перед запятыми, но ДОЛЖЕН быть пробел после каждой запятой.
  • Аргументы со значениями по умолчанию ДОЛЖНЫ располагаться в конце списка (после аргументов без значений по умолчанию).
  • Ключевые слова abstract и final, в случае их наличия, ДОЛЖНЫ располагаться перед указанием области видимости.
  • Ключевое слово static, в случае его наличия, ДОЛЖНО располагаться после указания области видимости.
  • В коде вызова функций и методов НЕ ДОЛЖНО быть пробела между именем функции или метода и открывающей круглой скобкой, НЕ ДОЛЖНО быть пробела после открывающей круглой скобки, НЕ ДОЛЖНО быть пробела перед закрывающей круглой скобкой.
  • В списке аргументов НЕ ДОЛЖНО быть пробелов перед запятыми, но ДОЛЖЕН быть пробел после каждой запятой.
  • Список аргументов МОЖЕТ быть разделён на несколько строк, каждая из которых дополнена слева одним отступом. В таком случае первый элемент списка аргументов ДОЛЖЕН начинаться с новой строки, и в каждой строке ДОЛЖЕН быть указан только один аргумент.
  • Желательно указывать типы параметров и возвращаемого значения

Конструкции if, elseif и else

  • Конструкция if выглядит следующим образом. Обратите внимание на круглые скобки, пробелы и фигурные скобки, а также на тот факт, что слова else и elseif располагаются в той же строке, что и закрывающая фигурная скобка предшествующего тела конструкции.
  • Ключевое слово elseif СЛЕДУЕТ использовать вместо отдельного сочетания else и if. Так конструкция будет представлять собой одно слово.
<?php
 
if ($expr1) {
    // тело if
} elseif ($expr2) {
    // тело elseif
} else {
    // тело else
}

Конструкции switch и case

  • Конструкция switch выглядит следующим образом. Обратите внимание на круглые скобки, пробелы и фигурные скобки. Выражение case ДОЛЖНО быть смещено на один отступ от switch, а ключевое слово break (или иное слово, обозначающее выход из конструкции) ДОЛЖНО располагаться на том же уровне отступов, что и тело case. В том случае, когда в непустом теле case умышленно не используется break, ДОЛЖЕН быть комментарий в стиле // no break.
<?php
 
switch ($expr) {
    case 0:
        echo 'First case, with a break';
        break;
    case 1:
        echo 'Second case, which falls through';
        // no break
    case 2:
    case 3:
    case 4:
        echo 'Third case, return instead of break';
        return;
    default:
        echo 'Default case';
        break;
}

Конструкции while и do while

  • Конструкция while выглядит следующим образом. Обратите внимание на круглые скобки, пробелы и фигурные скобки.
<?php
 
while ($expr) {
    // тело конструкции
}
  • Соответственно, конструкция do while выглядит следующим образом. Обратите внимание на круглые скобки, пробелы и фигурные скобки.
<?php
 
do {
    // тело конструкции
} while ($expr);

Конструкция for

  • Конструкция for выглядит следующим образом. Обратите внимание на круглые скобки, пробелы и фигурные скобки.
<?php
 
for ($i = 0; $i < 10; $i++) {
    // тело for
}

Конструкция foreach

  • Конструкция foreach выглядит следующим образом. Обратите внимание на круглые скобки, пробелы и фигурные скобки.
<?php
 
foreach ($iterable as $key => $value) {
    // тело foreach
}

Конструкция try catch

  • Блоки конструкции try catch выглядят следующим образом. Обратите внимание на круглые скобки, пробелы и фигурные скобки.
<?php
 
try {
    // тело try
} catch (FirstExceptionType $e) {
    // тело catch
} catch (OtherExceptionType $e) {
    // тело catch
}

Операторы

  • Перед и после бинарного оператора ДОЛЖЕН стоять пробел, за исключением операции конкатенации (.)
<?php
 
$a = $b + $c;
$a = 'string 1'.'string 2';

Переменные

  • Переменные ДОЛЖНЫ именоваться в under_score (английские слова в идентификаторах разделяются подчеркиваниями).

Для автоматического контроля за соблюдением стиля кода используется php-cs-fixer.
