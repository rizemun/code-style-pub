Использование чек-листов при тестировании задач

Чек-лист - технический документ, который представляет собой набор проверок(грубо говоря "тест-кейсов") для проверки определенной задачи.

Составление чек-листов

Что должен включать в себя чек-лист:

1. Дату создания чек-листа.
2. Дату обновления чек-листа.
3. Окружение(ОС, браузер, версия браузера) на котором проводится тестирование.
4. Информация о том, кто проводил тестирование.
5. Список проверок по конкретной задаче.
6. Фактический результат(Pass, Fail, Skip) по каждому пункту в чек-листе.
7. Время фактического прохода по ЧЛ.
8. Дату фактического прохода по ЧЛ.
9. Ссылка на тестируемую страницу/задачу.
10*. Ссылка на макет.

По итогу прохождения в чек-листе остается запись о фактическом проходе чек-листа, который включает в себя:

  1. Время и дата прохода.
  2. Окружение на котором проходило тестирование.
  3. Результат проверок по каждому пункту.
  4. Указание, кто проводил тестирование по каждому пункту.

Шаблон чек-листа

Шаблон чек-листа по верстке

(!) Каждый новый проход по чек-листу добавляется отдельно не удаляя/редактируя прошлый проходы.

К пунктам, которые были пропущены(статус Skip), нужно добавить комментарий с указанием причины пропуска.

К пунктам, по которым были ошибки(статус Fail), нужно добавить комментарий с указанием номера/ссылкой на карточку бага.
